# pybastion-integration

`pybastion-integration` runs PyBastion's integration analysis pipeline.

It consumes unit inventory artifacts produced by `pybastion-unit` and generates
integration seam specifications. These specifications are intended for review,
validation, and agent-assisted integration test generation.

## Command

Run integration analysis:

    pybastion-integration \
      --target-root /path/to/project \
      -v

Run integration analysis with graph checking:

    pybastion-integration \
      --target-root /path/to/project \
      --check-graph \
      -v

Using the unified PyBastion config:

    pybastion-integration \
      --target-root /path/to/project \
      --config pybastion_config.toml \
      --check-graph \
      -v

The same pipeline can also be run through the root CLI:

    pybastion integration \
      --target-root /path/to/project \
      --config pybastion_config.toml \
      --check-graph \
      -v

For the full unit and integration workflow:

    pybastion all /path/to/project \
      --config pybastion_config.toml \
      --readiness \
      --check-graph \
      -v

## Pipeline

The integration pipeline currently runs three stages:

1. Build the execution instance call graph
2. Generate integration test specifications
3. Split specifications by source unit and target unit pair

The graph checker can optionally run after Stage 1. It validates that the
generated graph lines up with the unit inventory artifacts.

## Required Inputs

`pybastion-integration` expects unit inventory artifacts generated by
`pybastion-unit`.

By default, it reads inventories from:

    dist/pybastion/inventory/

The normal workflow is:

1. Run `pybastion-unit` against the target project.
2. Run `pybastion-integration` against the same target project.
3. Review or consume the generated integration specifications.

## Outputs

By default, integration analysis writes under:

    dist/pybastion/integration-output/

Typical outputs include:

    dist/pybastion/integration-output/
      stage1-ei-cfg.pkl
      inventory-graph-report.yaml
      stage2-integration-test-specs.yaml
      specs/

The split files under `specs/` are grouped by source unit and target unit.
They are intended to be small enough for review and downstream agent use.

## Configuration

`pybastion-integration` can run with internal defaults or with the unified
PyBastion config file.

Generate the config from the root CLI:

    pybastion config init

Then pass it to the integration pipeline:

    pybastion-integration \
      --target-root /path/to/project \
      --config pybastion_config.toml \
      --check-graph \
      -v

The integration pipeline reads the `[integration]` section of the unified
config.

## Relationship to pybastion-unit

`pybastion-unit` produces the inventory artifacts consumed by
`pybastion-integration`.

`pybastion-integration` does not reanalyze source units directly. It works from
the unit inventory and builds integration-level artifacts from that structured
model.
